Coco is a premium travel-planning experience built as a full-stack MERN application with a warm, luxury-inspired brand identity.

Project identity
- App name: Coco
- Tagline: The Trip Planner
- Purpose: Help users plan trips with a calm, polished, and highly guided experience.

Architecture summary
- Frontend: React (Create React App) with Tailwind CSS, Framer Motion, React Router, Axios, React Hot Toast, React Icons.
- Backend: Node.js + Express with MongoDB + Mongoose.
- Auth: JWT-based authentication with bcrypt password hashing, Google OAuth integration, and protected routes.
- Maps and planning: backend routes and controllers handle trip planning, map-related requests, and AI-assisted itinerary generation.

Important setup notes
- The frontend lives in client/ and is started with npm start.
- The backend lives in server/ and is started with npm run dev.
- Environment variables must be loaded from the server .env file using dotenv at the top of server/server.js.
- MongoDB is expected to connect through Atlas in this project.

How the app works
1. The browser loads the React frontend.
2. React screens call backend endpoints via Axios.
3. Express routes delegate to controllers.
4. Controllers interact with the MongoDB models.
5. The response is sent back to the frontend and rendered in the UI.

Auth / user flow
- Users can register and log in via email/password or Google auth.
- New signups receive a one-time verification code that is sent to the email address on file.
- A dedicated verification screen guides users through email confirmation before they continue deeper into the app.
- Authenticated users can create trips, review saved plans, view maps, and manage profile data.
- Protected routes ensure only authenticated users can reach private screens.

# 7. Role of Express

```

React calls these APIs.

# 8. Role of Mongoose

Mongoose sits between Express and MongoDB.

Instead of writing raw MongoDB commands, you can do:

```javascript
const user = await User.findOne({ email });
```

instead of low-level database operations.

## One correction

You said:

> "Node.js connects frontend to backend to db"

A more accurate statement is:

> **React (frontend) sends requests to the Express backend running on Node.js. Express processes those requests, uses Mongoose to interact with MongoDB, and returns the response back to React.**

That is the standard architecture used in most MERN applications.
